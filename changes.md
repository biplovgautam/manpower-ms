> Note: Document uploads remain local storage as requested (no Cloudinary changes).

## Environment variable notes

Use the following variables (see `client/.env.local.example`):

- `NEXT_PUBLIC_FILE_BASE_URL` → `https://api.manpower.biplovgautam.com.np`

### Backend (Render)
See `server/.env.example` for production template. Key values:
- `CLIENT_URL` → `https://manpower.biplovgautam.com.np`
- `MONGO_URI` → MongoDB connection string
- `JWT_SECRET` → strong secret
- `EMAIL_USER`, `EMAIL_PASS` (Gmail app password)
- `TWILIO_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_NUMBER`
- `PUBLIC_API_URL` → `https://api.manpower.biplovgautam.com.np` (optional, for logs/links)

## Render deployment steps (backend)
1. Create a new **Web Service** in Render from this repo.
2. **Root Directory**: set to `server` (recommended for this monorepo).
3. **Build Command**:
   - `pnpm install`
   - `pnpm build`
4. **Start Command**:
   - `pnpm start`
5. Add environment variables from `server/.env.example` in Render.
6. Set **Auto Deploy** to enabled.
7. Add a custom domain in Render: `api.manpower.biplovgautam.com.np`.
8. Ensure CORS is configured with `CLIENT_URL=https://manpower.biplovgautam.com.np`.

If you prefer keeping the repo root in Render instead of `server`, use:
- Build: `pnpm --filter server install && pnpm --filter server build`
- Start: `pnpm --filter server start`
